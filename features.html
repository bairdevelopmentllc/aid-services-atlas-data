Features built for trust and reliability

ASA is designed to work without accounts, without ads, and without collecting personal data. It is “local-first” by default after the initial download.

Offline-first by design

  • Downloads needed data after location confirmation
  • This data stays on device for offline access
  • Data will update when next online

County-equivalent location model

  • Uses coarse GPS and reverse geocoding to determine an admin area
  • Stores an adminAreaId plus an optional fuzzy label (for example “near 22554”)
  • Does not store precise GPS coordinates

Layered data composition

Providers are defined in three layers, then merged at runtime:

  • Federal applies everywhere
  • State applies to a state, optionally limited to selected admin areas
  • Local applies to a single admin area

Search for discoverability

Search is not case-sensitive and includes text fields such as name, services, description, notes, and eligibility.

Neutral results and transparent limitations

  • No sponsored rankings, ads, or paid priority placement
  • Clear disclaimers and non-endorsement stance
  • Sorting is based on scope and clarity, not promotion
